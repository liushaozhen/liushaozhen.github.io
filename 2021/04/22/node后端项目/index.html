4、处理分页

使用 mongoose-aggregate-paginate-v2。支持多表关联的分页插件

5、用户权限校验

在入口处校验用户的token

6、关于数据传输安全问题

前端接口所有数据使用base64加密传给接口，后端接到数据后先解密再使用。 每次接口请求都做用户权限校验。这里还有很多不足，比如csrf、xss等问题未处理

7、规范化接口返回数据

作为前端，深知这个问题的重要性。所以在写后端项目的时候也处理了这个问题

  • 返回数据格式
  • 接口code规范
值 说明
0 成功
3006 token校验失败
4001 一般数据错误
5001 数据库错误

8、项目涉及到定时任务

这里没做太多的规范，在服务启动时执行一次，然后设置了定时器：每过24小时执行一次。这里可以使用 node-schedule 插件优化

9、进程守护

由于nodejs的单线程的脆弱性，一旦遇到运行错误等问题很有可能会严重到退出node进程导致系统或应用瘫痪。所以这一个是很有必要的。可以使用 pm2做node进程管理
