原型链

  • f.__proto__ = Foo.prototype
  • Foo.__proto__ = Function.prototype
  • Object.__proto__ = Function.prototype
  • Function.prototype.__proto__=Object.prototype
  • f.prototype.constructor = f
  • Foo.prototype.constructor = Foo
  • Function.prototype.constructor = Function
  • Object.prototype = null

数据类型

  • 基本数据类型：Number，String，Boolean，undefined，null，Symbol，bigInt，Object
  • 指针类型：Date，Function，Array，RegExp，Set，Map

类型判断

  • typeof
    • 不能判断null，object，date，array，这些判断出来都是Object
  • constructor
    • 主要利用原型上的 prototype.constructor中心实例进行判断
    • null和undefined是无效对象，不会有constructor
    • 函数的constroctor 是不准确的，可以重写
  • instanceof
    • 判断当前实例的原型与构造函数的原型是否为同一个对象
    • 实例的__proto__ 等于 构造函数的 prototype 则为true
    • 不够准确，可以通过__proto__一直往上找，例如 [] instanceof Array 和 [] instanceof Object 都为true，因为 Array也是Object
  • Object.prototype.toString.call
    • 返回值是 [object 类型] 例如 [object String]

let,const,var区别

  • var的作用域是整个封闭函数。let,const作用域在块级中
  • var有变量提升（不论var声明在第一行，都会提升到作用域最顶部），let不会。
  • var 允许重复声明，let不允许
  • 只要块级作用域内存在let，它所声明的变量就会绑定在这个块级作用域内
  • const用来声明常量，和let的区别就是不可变，声明时必须初始化
  • const obj = { a:1 }; obj.a = 2,不会报错，因为obj是指针类型，const存储的是obj的地址

普通函数和箭头函数的区别

  • 箭头函数没有原型链
  • 箭头函数是匿名函数，没有constructor，不能用new
  • 箭头函数没有arguments，可以用解构赋值代替
  • 箭头函数的this是定义的时候确认的，普通函数的this是执行的时候确定的

this指向问题

  • 匿名函数的this永远指向window
  • es5中，this执行最后调用它的那个对象。如果setTimeout，没有改变this，而且不是箭头函数，那么this指向window
  • 箭头函数this指向定义时的this，并不能通过call,apply,bind改变
  • new实例化后this指向实例
  • .前面是谁，this就指向谁
