http基础知识

http常用Header

通用标头 General

  • Request URL： 地址
  • Request Method：方法
  • Status Code：状态码

请求headers Request Headers

  • Accept：支持的格式
  • Cache-Control：控制缓存
  • Content-Type：请求的数据格式数据格式
  • cookie：cookie
  • Host：服务器域名
  • Referer：网页从哪里来的
  • Origin：类似Rerfer
  • User-Agent：设备信息

响应headers Response Headers

  • Access-Control-Allow-Origin：允许访问的来源
  • Date：时间
  • Content-Type：响应的数据格式

Request Payload

post请求会有的响应数据

http常见状态码

  • 1xx
    • 101：在http升级为websocket的时候，如果服务器同意变更，就会发送状态码101
  • 2xx
    • 200：成功
    • 204：成功，与200类似，但是没有body数据
    • 204：部分内容，通常在分块下载和断点续传中使用
  • 3xx
    • 301：永久重定向，例如jd之前的网址为360buy，有一段时间是永久重定向。浏览器会缓存
    • 302：临时重定向，例如登陆超时，重定向到登陆页
    • 304：当写上缓存命中时会返回这个状态码
  • 4xx
    • 400：一般为参数错误
    • 403：服务器禁止访问
    • 404：资源未找到
    • 405：方法不允许
  • 5xx
    • 500：服务器错误，没有具体错误信息
    • 502：服务器正常，但是访问出错，例如后端代码出问题
    • 503：服务器繁忙无

常见的请求方法

  • get：通常用来获取资源
  • head：获取资源的原信息
  • post：提交数据
  • put：修改数据
  • delete：删除数据
  • connect：建立连接隧道
  • options：列出可对资源实行的请求方法
  • trace：追踪请求

get和post有什么区别？

  • 参数角度：get参数在url中，post在请求体中，post相对安全
  • 缓存角度：get请求会被浏览器主动缓存，post默认不会
  • TCP角度：get会吧http、header、data一并发出去。post会先发送header，服务器响应100 continue，再发送data。（火狐浏览器除外，火狐的post只发送一个tcp包）

浏览器缓存策略

强缓存和协商缓存可以同时存在，强缓存优先级要搞

强缓存

规则：

  • 客户端——浏览器缓存——有缓存——直接返回
  • 客户端——浏览器缓存——没有缓存——服务器返回——客户端接收——缓存到浏览器

一旦资源命中强缓存, 浏览器便不会向服务器发送请求, 而是直接读取缓存. Chrome下的现象是 200 OK (from disk cache)；

  • cache-control 优先级最高。
    • public 资源被客户端和代理服务器缓存
    • private 资源仅被客户端缓存，代理服务器不缓存
    • no-store 请求和响应都不缓存
    • no-cache 使用协商缓存来验证
    • max-age 设置缓存时间，在指定时间后缓存过期，单位秒
  • Expires 缓存数据到期时间。当再次请求时的时间小于此时间，则直接使用缓存

协商缓存

规则

  • 客户端——浏览器缓存获得缓存标识——服务器验证是否失效——未失效——浏览器缓存获得数据
  • 客户端——浏览器缓存获得缓存标识——服务器验证是否失效——已失效——服务器返回——客户端接收——缓存到浏览器
  • ETag：服务器响应时，通过此字段告诉浏览器当前资源的唯一标识，优先级高于Last—Modified
    • If-None-Match：再次请求服务器时，包含此字段，使用此字段和Etag比较。不同说明资源改动过，重新响应整个资源200，相同直接走浏览器缓存304
  • Last-Modified：用于标记请求资源的最后一次修改时间
    • If-Modified-Since：缓存校验字段，和Last-Modified比较，相同走缓存304，不同走服务器200
    • If-Unmodified-Since：从某个时间算起，资源是否没有被修改，没有修改则走服务器200，修改了返回412预处理错误

主动缓存

通过 Cookie、sessionStorage，localStorage主动缓存
