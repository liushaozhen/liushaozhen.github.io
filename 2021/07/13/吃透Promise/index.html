吃透Promise

Promise基础知识

  1. Promise 构造函数立即执行，then异步执行
  2. Promise 状态一旦改变，就不会再变
  3. Promise 链式调用，return问题
  4. Promise 的then、catch方法参数如果不是函数，会发生值穿透
  5. Promise 异常处理
  6. Promise all,allSettled,race
  7. 中断Promise的执行

1、立即执行

Promise 构造函数是同步执行的，promise.then 中的函数是异步执行的。promise是微任务

2、Promise 状态

  • 有三种状态，pending，fullfilled（resolved），rejected
  • 状态一旦一旦改变，就不可变

3、链式调用，return问题

  • promise 每次调用 .then 或者 .catch都会返回一个新的promise，从而实现了链式调用
  • p.then.then 第一个then的 return 为第二个then的参数值。如果第一个then没有return，那么第二个then的参数值为undefined。 第一个then如果return Promise也同上。因为promise是通过返回一个新的promise来实现的链式调用
  • p.then；p.then 不是链式调用，两个then都可以拿到resolve的值，参考第二个示例
  • 返回值不能是promise本身，否则会死循环

4、值穿透

.then 或者 .catch 的参数期望是函数，传入非函数则会发生值穿透

5、异常处理

  • then中的错误会被下一个catch捕获
  • then中return new Error(‘错误’) 时，因为“promise的链式调用是通过返回一个新的promise实现的”，相当于 return Promise.resolve(new Error(‘错误’))，所以相当于return一个resolved的promise，会被下一个then捕获

6、其它方法

Promise.all()

  • 参数为Promise数组
  • 当所有Promise成功后，执行then
  • 当有一个Promise失败，则立即失败

Promise.allSettled()

  • 参数为Promise数组
  • 返回所有Promise的执行结果，无论成功或失败，都执行then

Promise.race()

  • 参数为Promise数组
  • 当有任何一个Promise成功或失败，就返回这个Promise的结果

7、中断Promise执行

Promise 有个缺点就是一旦创建就无法取消，所以本质上 Promise 是无法被终止的。但是可以通过以下两种方法中断

1.再某个 then/catch 之后，不再继续执行

在 then/catch 的最后一行返回一个永远在 pending状态的 Promise

2. 利用 Promise.race()

在合适的时候把Promise给reject掉。例如设置网络超时

手写promise

1、简单的promise

  • 构造函数立即执行
  • promise有三种状态 pending,fulfilled,rejected，默认是 pending
  • promise有一个value保存成功状态的值，一个reson保存失败状态的值
  • 有then方法，接受两个参数，分别是成功回调onFulfilled（传value）和失败回调（传reson）
  • 如果then抛出异常，那么把这个异常作为参数，传给下一个then

2、处理 构造函数中传入异步操作的问题

  • 我们需要先将成功和失败的回调分别存放起来，在executor()的异步任务被执行时，触发 resolve 或 reject，依次调用成功或失败的回调。

3、then的链式调用

为了达成链式，我们默认在第一个then里返回一个promise。秘籍规定了一种方法，就是在then里面返回一个新的promise,称为promise2：promise2 = new Promise((resolve, reject)=>{})

  • 将这个promise2返回的值传递到下一个then中
  • 如果返回一个普通的值，则将普通的值传递给下一个then中
  • 第一个then的返回值规定为 x ，判断x的函数叫做 resolvePromise

4、完成resolvePromise函数

  • 如果x === promise2，则是会造成循环引用，自己等待自己完成，则报“循环引用”错误
  • x不能为null
  • x是普通值，直接resolve(x)
  • 当x是对象或函数，生命then，如果取then报错，则走reject
  • 如果then是函数，用call执行then，第一个参数是this即x，后边是成功和失败回调
  • 如果成功回调还是promise，就递归继续机械
  • 成功和失败只能调用一个，所以设置called来防止多次调用

5、处理then异步、值穿透、成功回调函数合规性判断

  • onFulfilled,onRejected 都是可选参数，如果他们不是函数，必须被忽略
  • onFulfilled,onRejected 都是异步的，这里使用 setTimeout 解决异步
  • 如果onFulfilled或onRejected报错，则直接返回reject()
