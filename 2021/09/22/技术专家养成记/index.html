type vs interface

参考：Differences Between Type Aliases and Interfaces
​

相同点

1. 都可以描述对象或者函数

2. 都可以扩展

不同点

Type aliases and interfaces are very similar, and in many cases you can choose between them freely. Almost all features of an interface are available in type, the key distinction is that a type cannot be re-opened to add new properties vs an interface which is always extendable.
​

翻译：类型别名和接口非常相似，在许多情况下，您可以在它们之间自由选择。接口的几乎所有功能都可以在类型中使用，关键区别在于类型无法重新打开添加新属性，而接口始终是可扩展的。

1. 扩展方式不同：

interface：通过extends扩展
type：通过交叉类型的方式扩展

2. 重复声明

Type aliases may not participate in declaration merging, but interfaces can.
翻译：类型别名不能参与声明合并，但接口可以。
interface 可以重复声明，多次声明会合并，type报错

3. 声明类型

Interfaces may only be used to declare the shapes of objects, not rename primitives.

翻译：接口只能用于声明对象的形状，不能重命名基本体。
​
type 可以声明基本类型别名，联合类型，元组等类型。而接口不能声明基本数据类型

4. 错误提示

Interface names will alwaysappear in their original form in error messages, but only when they are used by name.

翻译：接口名称将始终以其原始形式出现在错误消息中，但仅当它们按名称使用时。
在错误提示中，接口会直接指向原始类型。 但是type也是这样，官网的这个不同并没有太懂

5. 类型别名不能被 extends和 implements

总结

For the most part, you can choose based on personal preference, and TypeScript will tell you if it needs something to be the other kind of declaration. If you would like a heuristic, use interface until you need to use features from type.
​

翻译：大多数情况下，您可以根据个人喜好进行选择，TypeScript会告诉您是否需要其他类型的声明。若你们想要一个启发式的，那么在你们需要使用来自类型的特性之前，使用接口。
​

官网对这个没有严格要求，此处可以参考 ant-design-pro 规范
推荐任何时候都是用 type， type 使用起来更像一个变量，与 interface 相比，type 的特点如下：

  • 表达功能更强大，不局限于 object/class/function
  • 要扩展已有 type 需要创建新 type，不可以重名
  • 支持更复杂的类型操作

基本上所有用 interface 表达的类型都有其等价的 type 表达。在实践的过程中，我们也发现了一种类型只能用 interface 表达，无法用 type 表达，那就是往函数上挂载属性。
